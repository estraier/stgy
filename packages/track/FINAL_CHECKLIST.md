# stgy-track final checklist

This checklist is for the current TrackJSON / FIT viewer update.

Use it before committing or tagging the work.

## 1. Confirm changed and added files

Run:

```sh
git status --short
```

Expected changed or added files are in these groups.

### Source files

```text
src/activity.ts
src/analysis.ts
src/analysis.test.ts
src/browser.ts
src/css.d.ts
src/fit.ts
src/fit.test.ts
src/fit-demo.ts
src/geo.ts
src/geo.test.ts
src/gpx.ts
src/gpx.test.ts
src/index.ts
src/loader.ts
src/loader.test.ts
src/metadata.ts
src/metadata.test.ts
src/renderer.ts
src/renderer.test.ts
src/stgy-track.css
src/trackjson.ts
src/trackjson.test.ts
fit-demo.html
index.html
```

Notes:

```text
src/index.ts
  viewer bundle entrypoint
  imports Leaflet CSS and stgy-track CSS
  does not export FIT helpers

src/loader.ts
  accepts .trj / .json / .geojson / .trjgz by extension
  accepts application/geo+json+gzip and other +gzip MIME types

src/renderer.ts
  viewer rendering, HUD, graph, smoothing, download link

src/fit.ts
  FIT parsing
  TrackActivity
  uniform / aggregate downsampling
  TrackJSON output
  numeric precision

src/trackjson.ts
  pure TrackJSON parse / compact / downsample / metadata helpers
```

### Documentation and package files

## 2. Run automated checks

Run:

```sh
npm run test
npm run check
npm run lint
npm run bundle
```

Or run:

```sh
npm run verify
npm run bundle
```

Expected:

```text
All tests pass
TypeScript check passes
ESLint passes
Browser bundles are generated
```

## 3. Build distributable files

Run:

```sh
npm run dist
```

Expected:

```text
dist/index.js
dist/index.d.ts
dist/fit.js
dist/fit.d.ts
dist/trackjson.js
dist/trackjson.d.ts
dist/track-viewer.js
dist/fit-demo.js
```

If CSS output is generated by esbuild, also expect:

```text
dist/track-viewer.css
```

## 4. Check package contents

Run:

```sh
npm run pack:dry-run
```

Expected package contents include:

```text
package.json
README.md
TRACKJSON.md
CHANGELOG.md
LICENSE
trackjson.schema.json
dist/
```

Expected package contents do not need to include:

```text
TESTING.md
RELEASE.md
examples/
src/
```

Those can remain repository files only.

## 5. Check package exports

After `npm run dist`, confirm these files exist:

```sh
ls dist/index.js dist/index.d.ts
ls dist/fit.js dist/fit.d.ts
ls dist/trackjson.js dist/trackjson.d.ts
ls dist/track-viewer.js
ls dist/fit-demo.js
```

The package exports expect:

```text
stgy-track
stgy-track/fit
stgy-track/trackjson
stgy-track/trackjson.schema.json
stgy-track/track-viewer.js
stgy-track/track-viewer.css
stgy-track/fit-demo.js
```

If `dist/track-viewer.css` is not generated, either fix the bundle output or
remove `./track-viewer.css` from package exports.

## 6. Manual example check

Start the HTTP server from the package root.

```sh
npm run bundle
npm run serve:examples
```

Open:

```text
http://localhost:8000/examples/index.html
```

Check:

```text
simple-track.html
  raw .trj route renders

compressed-track.html
  .trjgz route renders

multi-track.html
  multiple track sources render

pin.html
  inline pins and popups render

graph.html
  graph and HUD render
```

Expected:

```text
No 404 for dist/track-viewer.js
No 404 for dist/track-viewer.css
No "StgyTrackViewer is not defined"
No "Track data MIME type is not supported"
No permanent "Map loading..."
```

## 7. Manual FIT demo check

Open the FIT demo through HTTP.

Use the actual path in the project.

Examples:

```text
http://localhost:8000/fit-demo.html
```

Check uploads:

```text
.fit
.trj
.json
.geojson
.trjgz
```

For each input type, check:

```text
route renders
summary is shown
downsampling disabled works
uniform downsampling works
aggregate downsampling works
Download TrackJSON produces .trj
compressed download produces .trjgz
numeric compaction is applied
```

## 8. Manual graph check

Use `examples/graph.html`.

Check:

```text
graph panel appears
X axis can switch distance / time / sample
Y series can switch altitudes / heartRates / cadences / powers / speeds / torqueEffectivenessPercentage / pedalSmoothnessPercentage
smoothing changes only the graph
HUD remains raw
Y axis starts at 0 for standard series
tick labels include units
```

## 9. Manual loader compatibility check

Check these sources if the project has examples for them.

```text
.trj with application/octet-stream
.trjgz by extension
data:application/json
data:application/geo+json+gzip
application/*+gzip
```

Expected:

```text
TrackJSON and gzip TrackJSON load correctly.
```

## 10. Security spot check

Use a small malicious sample manually or in a scratch file.

Check that these do not execute:

```text
javascript: links
HTML in title or description
inline event handlers
unsafe image URLs
unsafe data-download-src
```

Expected:

```text
untrusted content is escaped, ignored, or rejected
```

## 12. Final git diff review

Run:

```sh
git diff --stat
git diff -- package.json
git diff -- src/index.ts
git diff -- src/loader.ts
git diff -- src/renderer.ts
git diff -- src/fit.ts
git diff -- src/trackjson.ts
```

Look for:

```text
debug code
temporary files
accidental generated files
large sample files
unwanted console.log
old comments that contradict current behavior
```

## 13. Suggested commit split

If the diff is large, consider splitting into these commits.

```text
1. Add TrackJSON format helpers and schema
2. Add FIT parsing and TrackJSON conversion
3. Add viewer graph, HUD, and download features
4. Add FIT demo upload/download/downsampling support
5. Add examples and documentation
6. Package and release metadata cleanup
```

If you prefer a single commit, use a broad message:

```sh
git add .
git commit -m "Add TrackJSON viewer, FIT conversion, and examples"
```
