STGY Track Viewer Feature Demo

This page demonstrates the implemented features of STGY Track Viewer.

0. Minimal setup: loading with data-src only

This is the minimal setup using only data-src.
It verifies that TrackJSON / GeoJSON can be loaded directly from the figure element.

0.1. Compressed data URL: application/geo+json+gzip

This demo uses data:application/geo+json+gzip;base64,... directly in data-src.

1. Basic pins and popup sizing with full auto-fit

This map does not specify data-lat, data-lon, or data-zoom.
The view is automatically fitted so that all pins are visible with 50px padding.

2. Fully fixed view

This map specifies data-lat, data-lon, and data-zoom.
The viewer respects the author's intended view and does not auto-fit the map.

3. Routes and GeoJSON pins with fixed zoom

This map specifies only data-zoom="12".
It mixes an HTML-defined pin, Tokyo Station in purple, with a GeoJSON-defined pin, Senso-ji in gold. Both should open popups when clicked.

4. Guide map with data-render="pin"

This demo uses .stgy-track-sources a.track-source[data-render="pin"].
Initially, only a representative guide pin is shown. Clicking the guide pin toggles the corresponding route layer.

5. Initial view from route bounds

This map has no inline pins and does not specify data-lat, data-lon, or data-zoom.
The initial center and zoom are computed from all route coordinates so that the whole route from Tokyo Station toward Mount Takao fits in the view.

6. coordinateProperties overlay and graph

This demo displays times, distances, altitudes, heartRates, cadences, powers, and speeds attached to each coordinate of a LineString.
On desktop, hover over the route or graph to inspect the nearest sample. Click the route or graph to pin the sample, and click the map background to clear it.
On mobile, tap the route or graph to pin a sample, and tap the map background to clear it.

7. Automatic overseas map layer fallback

When the map center is outside Japan, the viewer does not include GSI map layers and instead uses CyclOSM / OpenStreetMap / OpenTopoMap.

8. Error display: unsupported MIME type

This demo specifies a text/plain data URL. Since the MIME type is outside the whitelist, an error message should be displayed on the map.

9. Error display: invalid JSON

The MIME type is application/geo+json, but the JSON payload is intentionally broken.

10. Error display: missing template

This demo specifies data-src="#missing-template". Since no matching DOM element exists, loading should fail with an error message.

11. GeoJSON popup media: multiple links and images

This demo renders multiple links and images from GeoJSON properties.links and properties.images in a safe popup.

12. Display toggles: disabling overlay or graph

data-show-overlay="false" disables only the bottom-left overlay while keeping the map sample marker and graph interaction enabled.
data-show-graph="false" disables the graph panel while keeping the map overlay and sample marker enabled.

13. Multi-route graph switching

This demo shows multiple LineString features, each with its own coordinateProperties.
Click a route to make it active and switch the graph to that route. Click or tap a route or graph to pin a sample, and click or tap the map background to clear the pinned sample.

14. Original data download link

This demo shows the optional original-data download link generated from data-download-src.
The viewer loads the digest track data from data-src, while the download link points to the original master data specified by data-download-src.
