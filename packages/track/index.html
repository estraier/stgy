STGY Track Viewer Feature Demo

これまでに実装された全機能の挙動を確認するテストページです。

0. 最小構成：data-src だけで読み込む

data-src だけを指定した最小構成のデモです。
TrackJSON / GeoJSON の読み込み元を figure に直接書けることを確認します。

0.1. 圧縮 data URL：application/geo+json+gzip

data:application/geo+json+gzip;base64,... を data-src に直接指定するデモです。

1. 基本機能：ピン描画とポップアップサイズ指定（完全オートフィット）

data-lat, data-lon, data-zoom をすべて指定していません。
自動的に50pxの余白を持って全体が収まるようにズーム計算されます。

2. 画角の完全固定（オートフィット無効）

data-lat, data-lon, data-zoom をすべて指定しています。
著者の意図を完全に尊重するため、マップは動きません。

3. ルートとGeoJSONピン（中心移動＋ズーム維持）

data-zoom="12" のみを指定しています。
この地図には、HTML側のピン（東京駅/紫）と、GeoJSON側のピン（浅草寺/金）の両方が混在して表示されます。どちらもクリックしてポップアップが開きます。

4. ガイドマップ：data-render="pin"

.stgy-track-sources a.track-source[data-render="pin"] のデモです。
最初は代表点だけがピンとして表示され、クリックすると対応するルート表示がトグルされます。

5. ルート範囲による初期画角

インラインピンを置かず、data-lat, data-lon, data-zoom も指定していません。
ルートの全座標をもとに中心とズームを決定し、東京駅から高尾山方面までの広いルート全体が収まることを確認します。

6. coordinateProperties HUD

LineString の各座標に対応する times, elevations, heartRates, cadences, powers, speeds を表示するデモです。
ルート上にマウスを乗せると、地図の左下に最寄り点の値がリスト形式で表示されます。

7. 海外地図の自動切り替えとフォールバック座標判定

8. エラー表示：非対応 MIME type

text/plain の data URL を指定しています。MIME ホワイトリスト外なので、地図上にエラーが表示されます。

9. エラー表示：壊れた JSON

MIME は application/geo+json ですが、JSON が壊れています。

10. エラー表示：存在しない template

data-src="#missing-template" を指定しています。該当する DOM 要素がないため、読み込みエラーになります。
